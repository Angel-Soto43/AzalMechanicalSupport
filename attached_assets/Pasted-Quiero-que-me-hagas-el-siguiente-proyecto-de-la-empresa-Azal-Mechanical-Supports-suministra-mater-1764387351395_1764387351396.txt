Quiero que me hagas el siguiente proyecto de la empresa "Azal Mechanical Supports", suministra materiales electromenicos te dare los requerimientos funcionales y no funcionales para que sepas que hacer en cada seccion del proyecto y tambien el objetivo para que te des la idea bien
OBJETIVO 
Desarrollar un gestor de archivos en la nube accesible vía web, que permita a la empresa de suministro de materia prima administrar, almacenar y consultar documentos relacionados con sus contratos y operaciones. El sistema debe ofrecer un control total al administrador jefe (Víctor Hernández), garantizar la integridad de los datos cargados por los trabajadores, cumplir con normas internacionales de seguridad de la información (ISO/IEEE) y permitir acceso desde cualquier dispositivo mediante credenciales corporativas. para la facilitacion del manejo de informacion (archivos)
REQUERIMIENTOS FUNCIONALES1.- Módulo de Autenticación y Gestión de Usuarios
RF-01. Autenticación personalizada.
El sistema debe permitir iniciar sesión únicamente con usuarios y contraseñas generados por el Administrador Jefe.No debe permitir acceso mediante Google, Facebook u otros proveedores de identidad externos.
RF-02. Gestión de usuarios.
Solo el Administrador Jefe (Víctor Hernández) puede: 
Crear nuevos usuarios.
Asignarles un nombre de usuario y contraseña.
Editar información de los usuarios.
Activar o desactivar cuentas.
Los usuarios regulares no pueden modificar ninguna información de cuentas.

2.- Modulo de Gestion de Archivos en la nube 
RF-03 Subida de archivos 
los usuarios podran subir archivos como Word,Excel, powerpoint, PDF, imagenes , archivos ZIp, etc.
El sistema debe permitir registrar 
* ID o folio del contrato u archivo  (obligatorio)
* Fecha y hora de subida 
* Nombre del usuario que subio el archivo 
* Tipo y peso del archivo 
RF-04. Descarga y visualización* Los usuarios podrán visualizar y descargar los archivos almacenados.* El sistema debe permitir previsualización cuando sea posible (PDF, imágenes, documentos soportados).RF-05. Edición de archivos* Los usuarios pueden subir nuevas versiones de un archivo.* El sistema debe mantener un historial o indicar la última versión cargada (opcional según alcance).RF-06. Integridad y bloqueo de edición* Un usuario no puede modificar ni editar archivos subidos por otro usuario.* Cada archivo queda bloqueado al usuario propietario para ediciones posteriores (propiedad basada en usuario)

3.- Modulo de Permiso del Administrador jefe 
RF-07. Panel de control exclusivo 
el administrador jefe debe tener acceso a un panel donde pueda 
* Ver todos los archivos cargados por todos los usuarios 
* Consultar 
quien subio el archivo 
fecha y hora exacta de subida 
tipo de archivo 
peso del archivo 
ID de contrato
* El administrador puede borrar archivos, los demas usuarios no.
RF-08 Buscador avanzado 
el administrador debe contar con un buscador capaz de filtrar archivos por: 
hora/fecha de subida 
id del archivo

4. Modulo de seguridad (Cumplimiento de iso / IEE)
RF-09 cumplimiento de normas 
el sistema debe cumplir medidas esenciales basadas en 
ISO/IEC 27001 (seguridad de la informacion)
ISO/IEC 27002 (Contorles de seguridad)
IEEE 12207 (ciclo de vida de software)
politicas de acceso seguro.
RF-10 encriptacion 
los archivos deben alamacenarse con cifrado seguro (AES-256 o equivalente)
la comunicacion entre cliente servidor debe usar HTTPS con TLS 1.2 o superior 
RF-11 Control de sesiones 
cerrar secion automatica despues de un tiempo de inactividad 
bloqueo despues de multiples intentos fallidos 
RF-12 Registro de auditoria 
el sistema debe registrar 
inicio de secion 
subida de archivos 
descargas 
intentos fallidos 
eliminaciones (solo administrador)
5. Modulo de accesibilidad y Compatibilidad 
RF-13. Acceso Multiplataforma
el sistema debe permitir acceso desde 
Computadoras (Windows, macOS, Linux)
dispositivos moviles (Android, iOS)
RF-14. Diseño responsivo 
La interfaz Grafica debe adaptarse automaticamente a distintos tamaños de pantallas
